Trips Routs

Method URL Body Description
GET domain/trips/perPage={x}&page={y}&sort={trip_atribute}&reverse={yes} - Get a list of all the trips.
x represents the amount of trips that will be displayed on the page, y - the page number, trip_atribute - the sort by specific attribute, and "yes" if you want to sort from large to small.
GET domain/trips/single/{trip_id} - Get one trip by its _id
GET domain/trips/search?s={name or info} - Get a list of the trips by name or info. It will present 10 results per page
GET domain/trips/search?s={name_or_info}&perPage={x}&page={y} - Get a list of the trips by name or info. It will present x results per page and will take you to page y
GET domain/trips/category/{category} - Get a list of the trips by category. It will present 10 results per page
GET domain/trips/category/{category}?perPage={x}&page={y} - Get a list of the trips by category. It will present x results per page and will take you to page y
GET domain/trips/prices?min={x}&max={y} - Shows only the trips that their price is less/equal x and greater/equal y
GET domain/trips/prices?min={x}&max={y}&perPage={z}&page={w} - Shows only the trips that their price is less/equal x and greater/equal y. It will present z results per page and will take you to page w
GET domain/trips/tripsByUserId/{userId} - Shows all trips that a spesific user created.
POST domain/trips {
"name": "nahal bokek",
"info": "nice river for children",
"category": "south",
"img_url": "https://www.pexels.com/photo/river/",
"price": 0
} 		Add a new trip to the data base. You need to send a token in the `Headers` for this query.
(x-api-key)
PUT domain/trips/{trip_Id} {
"name": "rosh hanikra",
"info": "nice view on the sea",
"category": "north",
"img_url": "https://www.pexels.com/photo/sea/",
"price": 34
} 		Update a trip in the data base. You need to send a token in the `Headers` for this query.
(x-api-key)
DELETE domain/trips/{trip_Id} - Delete a trip from data base by trip's _id. You need to send a token in the `Headers` for this query.
(x-api-key)

Users Routs

Method URL Body Description
GET domain/users/perPage={x}&page={y}&sort={user_atribute}&reverse={yes} - Get a list of all the users.
x represents the amount of users that will be displayed on the page, y - the page number, user_atribute - the sort by specific attribute, and "yes" if you want to sort from large to small.
GET domain/users/single/{userId} - Get one user by his _id.
POST domain/users {
"name":"yehuda",
"email":"yehuda@gmail.com",
"password":"123456"
} 		Add a new user to the data base
POST domain/users/login {
"email":"yehuda@gmail.com",
"password":"123456"
} 		User login. If the email and password are correct the server will respond with a token
DELETE domain/users/{userId} - Delete a user from data-base by user's _id.
